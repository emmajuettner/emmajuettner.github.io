Piet Tutorial

Posted 19 Apr 2019.

(Originally written as a tutorial for the Lake Forest College Computer Science Club.)

Hi all! Today we’ll be looking at an esoteric programming language called Piet. Esoteric programming languages are languages where the concept or aesthetic of the language is usually more important than the actual functionality or usability (some examples of esoteric languages that we’ve done past CS Club tutorials on are LOLCODE and Brainfuck). No programming experience is required for this tutorial, although it will be easier if you have an understanding of how a stack works.

Piet is pretty unique in that the program is an image. Literally, you write code by placing pixels of color on a canvas. It’s inspired by the works of Piet Mondrian, who was famous for his paintings involving blocks of bright color. Today we’ll be learning the basics of Piet!

Resources:

1) A good intro to the language written by its creator; it explains how the syntax works and has links to some examples

2) An online editor for Piet

3) An ASCII table to help you print out characters

Let’s start out by making our first program! Because of the way Piet works, there’s a lot of different ways to implement any given program (you can see some different examples of fancy Hello World programs in the first link above). I’ll show a very simple example program, but see if you can create your own variations!

Before we start, you’ll need to be familiar with what a stack is. If you aren’t familiar with this term, think about a stack of books on a table; you can put more books on the top, and take books off the top, but you can’t access books in the middle of the stack. That’s how a stack works in computer programming, too. We “push” things onto the top of the stack, and “pop” them off when we want to use them, but we’re always accessing the item at the top of the stack.

I started by listing out the instructions I wanted to perform. Let’s say I want to print “Hi”. These are the ASCII values I want to print out: 72 105 (you can find these values in the ASCII table linked above). So I need to write a series of instructions that will get me those numbers and then print them out as characters (again, there’s more than one way to do this). To start, I’ll write this in pseudocode, not an actual Piet program. Here’s what I came up with:

push 72
out(char)
push 105
push 1
pointer

Now we need to translate those instructions into pixels (or codels, as Piet calls them). The integers are represented by the number of codels in a particular block of color, and the commands are represented by changes of color. Read the first link above for a detailed explanation and list of commands.

You can see my program in the above image. Try running it for yourself! There are some codels that I just filled in more or less randomly, since they weren’t needed for the program.) See if you can understand how it works, and then try to make a full Hello World program on your own. Make sure to test your program frequently; it’s easy to get mixed up with this language. If you’re having trouble figuring out how the program flows from one codel to the next, start out by making a program that just goes from left to right. If you want more of a challenge, try designing a program that actually forms a picture of something!

That’s all for today! Have fun experimenting with Piet!
