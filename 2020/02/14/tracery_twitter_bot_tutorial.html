Tutorial - Tracery Twitter Bots

14 Feb 2020 - Emma Juettner

This tutorial teaches the basics of making generative Twitter bots using Tracery and Cheap Bots Done Quick. Originally presented at Lake Forest College Computer Science Club in 2019. [No Coding Experience Required!]

What are Twitter bots?

Bots on Twitter range from very simple to very complex. They can generate stories, poems, pieces of art, jokes, and more. If you want to learn more about the many types of bots out there, a good resource is Botwiki, which collects examples of bots along with tutorials, essays, and other resources. For this tutorial, we’ll focus on bots made with Tracery, a generative tool that’s easy for beginners to pick up. Here are a few examples of bots I’ve made with Tracery:

  • @unicode_faces - a bot that makes faces using Unicode characters
  • @XFilesPlotBot - a bot that comes up with new X Files plots
  • @cs_club_img_bot - a bot that tweets captioned images of my family’s cat

Tools

  • Tracery
  • Cheap Bots Done Quick (CBDQ)

You don’t have to install anything for this tutorial. It can all be done online!

Tracery Overview

Tracery is a tool built for doing basic text generation, MadLibs-style, from a list of lists that you create. (If you’re familiar with JSON, that’s what this is, but you don’t need to know what JSON is to use Tracery.) When you paste your Tracery code into Cheap Bots Done Quick and connect a Twitter account, CBDQ will automatically start generating tweets from your Tracery lists and posting them to Twitter on a schedule that you set. You can make your Tracery code as complex as you like, so there’s a lot of room to be creative.

The online Tracery editor has a helpful visualization tool for keeping track of the parts of your list and how they work together, so I recommend writing your code there rather than in another text editor. Alternately, you can use this alternate editor, which may be slightly easier to understand for beginners.

Saving Variables

At this point we’ve covered everything you need to make a very basic bot. But we can add a little more sophistication by saving variables for use later in the sentence, rather than selecting every word randomly. To store a variable, use this notation: [variableName:#variableListName#] at the beginning of the ‘origin’ list. This will not show up in the output, but it will randomly pick an item from variableListName and save it to variableName. Then, when you want to refer to that variable, use #variableName#.

Modifiers

You may want to modify certain variables by capitalizing them, putting ‘a’ or ‘an’ in front of them, or making them plural or past tense. You can do that by adding modifiers to the variables; for example, #name.capitalize# will pick a random item from the ‘name’ list and capitalize the first letter. You can also create your own modifiers if you know Javascript, but that’s beyond the scope of this tutorial. Here are some built-in modifiers:

  • #variable.capitalize# capitalizes the first letter of variable
  • #variable.capitalizeAll# capitalizes all letters of variable
  • #variable.a# adds an ‘a’ or ‘an’ in front of variable
  • #variable.s# makes variable plural
  • #variable.ed# makes variable past tense

Note: Some of the modifiers may not work properly with words that have irregular froms of the past tense/plural/etc, or if you are creating a bot in a language other than English.
